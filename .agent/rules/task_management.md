# タスク管理の補足事項

## 基本方針
- 開発ライフサイクルの全般（Issue 起票、ブランチ作成、実装、PR 作成）については、**`github-dev-flow` スキル** (`.agent/skills/github-dev-flow/SKILL.md`) の手順を最優先で遵守してください。
- 本ファイルでは、スキルの手順を補完するプロジェクト固有の事項のみを定義します。

## プロジェクトの「唯一の真実（Source of Truth）」

本プロジェクトでは、進捗を以下の2つのファイルで管理します。

1.  **`docs/progress/roadmap.md` (永続的)**:
    - プロジェクト全体のロードマップ、完了したフェーズの履歴、永続的な決定事項を記録します。
    - このファイルは PR マージ後も `main` ブランチに残り続けます。
2.  **`docs/progress/task.md` (一時的)**:
    - **`github-dev-flow` スキル** に従い、現在のブランチで行う具体的な実装手順と進捗を管理します。
    - **重要**: スキルの推奨に従い、PR 作成時に `task.md` を削除し、その作業に関する進捗履歴は GitHub Issue に委ねます。

## 作業中の進捗・中断（栞機能）
中断時や定期的な保存（`/save`）の際には、再開時のコンテキストを完璧に復元するための「栞（Checkpoint）」を残してください。
これは、スキルの「3. 実装と進捗管理」を拡張するものです。

- **GitHub Issue へのコメント**: 実装の進捗と「次の一手（具体的アクション）」をコメントとして残してください。
- **task.md への記録**: 削除前の `task.md` に「決定事項・メモ」セクションを設け、中断時の詳細なコンテキスト情報を追記してください。
  - 書くべきこと：どのクラスのどの修正まで終わったか、次は何のコマンドを打つべきか、デバッグ中ならどの変数が怪しいか、など。

## クリーンアップ
- ユーザーから PR のマージ完了が報告されたら、**`github-dev-flow` スキル** の「6. クリーンアップ」に従って後片付けを実行してください。
