---
description: 進捗を保存し、次回再開時のための「栞（しおり）」を Issue と task.md に残します
---

1. `github-dev-flow` スキルの「3. 実装と進捗管理」に従い、現在のステータスを `docs/progress/task.md` に反映する。
2. 現在の作業コンテキスト（編集中のファイル、行数、未解決の課題等）を分析し、再開用の**「栞（Checkpoint）」**を作成する。
3. 作成した栞を、関連する **GitHub Issue にコメント**として投稿する。
4. 同様の栞を `task.md` の「決定事項・メモ」欄の末尾に「Checkpoint: [日付]」として追記する。
5. ユーザーに保存完了を報告し、プッシュが必要な場合はその旨を伝える。
